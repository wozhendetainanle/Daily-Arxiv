## 低算力可改进的 10 篇：每篇 3 个方法级改进点

下面保留 10 篇候选论文。每篇给 3 个对方法本身的改进方向，而不是泛泛复现或只换数据集。事实部分来自论文摘要/PDF 首页；改进方案、实验设计、收益风险为推断。

### 1. [SeqLoRA](https://arxiv.org/abs/2605.22743)：多概念 LoRA 的干扰控制

原方法事实：SeqLoRA 用 bilevel optimization 联合优化 LoRA 两个因子，并用正交适配降低多概念持续学习中的表示干扰。

改进点 1：干扰感知的动态 rank/正交预算。为每个概念-层对维护干扰分数 `I(layer, concept)`，由历史 LoRA 子空间夹角、旧概念验证 loss drift、cross-attention 对旧概念 token 的注意力泄漏组成；高干扰层增加正交惩罚或临时扩 rank，低干扰层压缩 rank。低算力实验可用 Stable Diffusion 1.5、10 个概念、`r in {2,4,8}`，比较固定 rank、统一强正交、动态 rank+正交。预期收益是更少参数下减少 attribute interference；风险是代理指标可能把 rank 分配带偏。

改进点 2：token 级概念路由，而不是整层共享同一 LoRA 混合。给每个概念维护一个轻量 gating head，根据 prompt token 和 cross-attention map 决定当前 token 激活哪些 LoRA 子空间，避免多个概念在无关 token 上互相污染。低算力实验可只在 U-Net cross-attention 层加 gate，测试双主体 prompt 的 identity preservation 和 binding error。预期收益是改善多概念组合；风险是 gate 学到 prompt 模板偏差，泛化到自由文本时不稳定。

改进点 3：加入反事实组合损失。构造 `A+B`、`A without B`、`B without A`、属性交换 prompt，让模型在多概念组合时保持概念独立性；损失可用 CLIP/DINO identity consistency 加 attention disentanglement。低算力实验只需为每对概念生成少量反事实 prompt，不增加训练图像。预期收益是降低属性串扰；风险是 CLIP/DINO 对细粒度身份不够敏感，可能优化到指标而非视觉质量。

### 2. [One LR Doesn't Fit All](https://arxiv.org/abs/2605.22297)：layerwise learning rate 控制

原方法事实：论文用 Heavy-Tailed Self-Regularization 衡量 Transformer 层权重谱，并据此为不同层分配学习率。

改进点 1：把 HT-SR 扩展成“谱状态 + 梯度噪声”的双信号控制器。令 `lr_l = base_lr * f(alpha_l) * g(snr_l)`，其中 `alpha_l` 是 heavy-tail 指标，`snr_l` 是最近若干 step 的梯度信噪比或方向一致性；谱上欠成熟但梯度噪声大的层不盲目放大学习率。低算力实验用 60M-150M GPT 在 TinyStories/OpenWebText 子集训练 5k-20k steps。预期收益是减少 loss spike 和低精度训练过冲；风险是新增控制器超参吃掉调参收益。

改进点 2：把层级学习率改成模块级学习率。Transformer 同一层里的 attention projection、MLP up/down projection、LayerNorm 参数谱结构不同，统一层级 LR 仍粗糙；可为模块组分别计算谱漂移和更新范数，形成 module-wise LR。低算力实验固定总 token budget，对比 uniform LR、layerwise LR、module-wise LR。预期收益是更好利用 FFN 容量；风险是模块粒度过细后统计噪声变大。

改进点 3：做 LoRA/QLoRA 迁移版 LLR。原方法面向全参训练，但 PEFT 中可调参数只是一小部分；可用基座权重 HT-SR 决定每层 LoRA adapter 的 LR 或 rank-LR 联动。低算力实验在 1B 以下模型上做 GSM8K/Alpaca 子集微调，比较 uniform adapter LR、HT-SR adapter LR、HT-SR rank+LR。预期收益是给 PEFT 一个低成本训练 recipe；风险是基座权重谱不一定能代表 adapter 的可塑性。

### 3. [DualOptim+](https://arxiv.org/abs/2605.21539)：LLM unlearning 的 optimizer state 设计

原方法事实：DualOptim+ 用 base state 捕获忘记/保留目标共享表示，用 delta states 保存目标特异残差，并提出 8-bit 变体降低内存。

改进点 1：把共享/解耦状态从固定结构改为冲突门控结构。用 forgetting gradient 与 retaining gradient 的 cosine conflict 决定某个参数块走 base state、delta state 还是混合 state；冲突低时共享，冲突高时解耦。低算力实验可在 TOFU 或 WhoIsHarryPotter 小模型 unlearning 上记录遗忘率、保留集困惑度和冲突分布。预期收益是减少过度解耦导致的效用损失；风险是梯度冲突估计对 batch 采样敏感。

改进点 2：在 optimizer update 中加入 retain-set Fisher trust region。对保留集重要参数限制 delta state 更新幅度，近似 EWC，但把约束放进 optimizer 而不是额外 loss。低算力实验用 1B 以下模型，小批量估计 diagonal Fisher，对比普通 DualOptim+ 和 Fisher-gated DualOptim+。预期收益是更稳地保持通用能力；风险是 Fisher 估计不准时会保护错误参数。

改进点 3：8-bit delta state 做冲突自适应精度。不是所有 delta state 都需要同样量化位宽；高冲突参数块保留 8-bit 或 16-bit，低冲突块降到 4-bit 或稀疏更新。低算力实验测同等显存下的 unlearning/retain Pareto frontier。预期收益是进一步降内存；风险是混合精度实现复杂，速度收益可能不如显存收益明显。

### 4. [LABO](https://arxiv.org/abs/2605.22054)：LLM 辅助 Bayesian optimization

原方法事实：LABO 把低成本 LLM 预测和真实实验观测放进同一 BO loop，用 gating 动态平衡 LLM prediction 与 expensive experiment。

改进点 1：把 LLM 预测拆成均值与校准不确定性，而不是只当 cheap evaluator。对 LLM 输出做 ensemble/self-consistency，估计 epistemic uncertainty，并把它作为 BO surrogate 的噪声项或 gating 输入。低算力实验用 Branin、Hartmann、分子性质 toy benchmark，比较无校准 LABO 与 calibrated LABO。预期收益是减少 LLM 高置信错误污染搜索；风险是 self-consistency 增加调用成本。

改进点 2：加入 multi-fidelity 层级：LLM、快速模拟、小规模真实实验。LABO 当前核心是 LLM vs experiment，可扩展成三层或多层 acquisition，只有在低层预测分歧大时才升级到高层。低算力实验可用合成函数模拟不同 fidelity 噪声，或材料 toy task 中用小模型代理中等 fidelity。预期收益是更贴近科学实验流程；风险是 fidelity 间偏差建模不当会让 BO 过早相信廉价层。

改进点 3：给 LLM proposal 加 novelty memory 和反事实约束。记录已探索区域、失败模式和 LLM 常见模板，惩罚重复建议；同时要求 LLM 生成“为何不是相邻点”的反事实解释，作为 diversity prior。低算力实验统计相同 budget 下 unique basin coverage 和 best-found value。预期收益是减少 LLM 在熟悉区域打转；风险是 novelty 过强会牺牲 exploitation。

### 5. [Why Semantic Entropy Fails](https://arxiv.org/abs/2605.21801)：policy optimization 的不确定性信号

原方法事实：论文指出 semantic entropy 在 policy optimization 中存在 anisotropic gap 和 calibration gap，并提出 GCPO 以几何感知与 reward calibration 改善不确定性信号。

改进点 1：把不确定性从 response-level 下沉到 token/span-level。语义不确定常集中在推理链的关键 span，而不是整条回答；可用 span embedding dispersion 与 reward attribution 结合，只对高不确定关键 span 降权或重采样。低算力实验用数学推理小模型离线 rollouts，比较 response-level GCPO 与 span-level GCPO。预期收益是更精细地保留有效学习信号；风险是 span attribution 本身噪声较大。

改进点 2：引入梯度方差预测器替代人工设计 uncertainty proxy。训练一个小回归器，用 prompt、response embedding、reward 分布预测该样本对梯度方差的贡献，再用于 policy update weighting。低算力实验可以离线收集小批次梯度范数作为标签，不需要完整 RL 训练。预期收益是直接优化论文关心的 gradient variability；风险是预测器可能过拟合当前模型阶段。

改进点 3：做校准温度的在线闭环控制。GCPO 需要 uncertainty 与 reward strength 对齐，可加入一个小的 controller，使高不确定样本的有效权重满足目标梯度方差区间。低算力实验在 GRPO-lite 或 offline DPO-style objective 上比较固定校准、线性校准、闭环校准。预期收益是跨任务更稳；风险是 controller 可能和 optimizer 动态相互震荡。

### 6. [What are the Right Symmetries for Formal Theorem Proving?](https://arxiv.org/abs/2605.22257)：形式化证明的 rewrite symmetry

原方法事实：论文用 rewriting categories 定义 proof equivariance 和 success invariance，并提出 test-time aggregation 缓解 LLM prover 对等价表述敏感的问题。

改进点 1：把均匀 rewrite ensemble 改成预算感知 rewrite bandit。每个 rewrite family 是一个 arm，例如变量重命名、交换律重排、目标式左右变换、lemma 引入、归纳变量替换；根据 Lean 中间反馈分配剩余采样预算。低算力实验在 MiniF2F/ProofNet 子集固定 16-64 次采样，比较原始采样、均匀 ensemble、bandit。预期收益是同预算提升 pass rate；风险是中间 reward 偏短视。

改进点 2：学习题面到 rewrite prior 的轻量映射。用题面符号特征、AST 深度、是否出现 commutative operator、是否需要 induction 等特征，预测应优先尝试哪类 rewrite。低算力实验用历史失败/成功日志训练 logistic 或小 MLP，不需要微调 LLM。预期收益是减少无效 rewrite；风险是训练集小导致 prior 偏向特定 benchmark。

改进点 3：把 rewrite 生成从语法等价扩展到 proof-state 等价。不是只改原始 theorem statement，而是在 Lean 返回 proof state 后对目标和上下文做等价重写，形成中途修复机制。低算力实验统计失败样本中二次 rewrite 后的 recovery rate。预期收益是改善长证明搜索；风险是 proof-state rewrite 可能引入更多编译开销。

### 7. [HyLoVQA](https://arxiv.org/abs/2605.22035)：动态 LoRA continual VQA

原方法事实：HyLoVQA 用 memory anchors 和 hypernetwork 动态生成 LoRA adapters，解决 continual VQA 中任务/对象干扰。

改进点 1：把 anchor memory 改成因果角色分解 memory。当前 anchor 偏视觉对象和文本任务，可进一步拆成 object、relation、attribute、question-intent 四类 anchor，让 hypernetwork 生成更结构化的 LoRA。低算力实验在 CLEVR 或小规模 VQA v2 stream 上做 compositional split。预期收益是增强组合泛化；风险是角色标注或自动解析错误会污染 memory。

改进点 2：加入 adapter contrastive routing。对同一图像不同问题、不同图像同一问题构造正负对，让生成的 LoRA adapter 在功能空间里可分而不过度分裂。低算力实验只需在现有 VQA batch 中重排 pair，比较 forgetting、current-task accuracy 和 adapter collapse 指标。预期收益是降低任务间误路由；风险是 contrastive loss 权重过大会损害共享知识。

改进点 3：扩展到 abstract visual reasoning 的低成本诊断头。给 HyLoVQA 加一个轻量 symbolic relation probe，要求 adapter 对颜色/形状/数量/位置关系有可读预测。低算力实验可用 CLEVR、PGM 小子集，不必训练大 VLM。预期收益是让方法从 VQA 走向抽象视觉推理；风险是 probe 提升不一定转化为最终问答准确率。

### 8. [Improved Adaptive PID Optimizer](https://arxiv.org/abs/2605.21968)：AdaPID/Adam 系列优化器

原方法事实：论文把 AMSGrad 的非递增有效学习率和 DiffGrad 的梯度差分调制并入 AdaPID，提出 IAdaPID-ADG。

改进点 1：把 PID 系数设为按层自适应，而不是全局固定。不同层的梯度延迟、噪声和曲率不同，P/I/D 三项可根据梯度自相关和局部 loss curvature 的 EMA 调整。低算力实验在 CIFAR、Tiny Shakespeare、小 ViT 上比较全局 PID 和 layerwise PID。预期收益是减少 overshoot；风险是 PID 参数过多导致调参复杂。

改进点 2：加入 decoupled weight decay 与 scale-invariant normalization。AdamW 的成功部分来自解耦权重衰减，AdaPID 系列若仍耦合正则可能不利于 Transformer；可把 IAdaPID-ADG 改成 AdamW 风格，并对更新方向做 layer RMS normalization。低算力实验在小 GPT/ViT 上比较 AdamW、原 IAdaPID-ADG、decoupled IAdaPID-ADG。预期收益是更公平地进入现代架构；风险是改动后不再能清晰归因于 PID。

改进点 3：做低精度稳定版 PID optimizer。PID 的差分项可能放大量化噪声，可对 D-term 使用 sign/clip 或 EMA-denoise，并把状态压到 8-bit。低算力实验模拟 bf16/8-bit optimizer states，观察 loss spike 和最终精度。预期收益是让方法适配显存受限训练；风险是 D-term 被过度平滑后失去控制意义。

### 9. [Optimizer-Induced Spectral Scaling Laws](https://arxiv.org/abs/2605.21803)：优化器与表征谱

原方法事实：论文比较 AdamW、Muon 等优化器如何改变 Transformer FFN 表征谱秩 scaling，指出 optimizer 是 representation scaling 的一条轴。

改进点 1：把谱秩观察变成训练时正则。若 Muon 的优势来自更高 hard-rank 利用率，可加入轻量 spectral spread regularizer，鼓励 FFN activation covariance 不过早塌缩。低算力实验训练 50M GPT，对比 AdamW、AdamW+spectral regularizer、Muon。预期收益是让 AdamW 获得部分 Muon 表征优势；风险是谱正则计算成本和稳定性。

改进点 2：研究 LoRA rank 与表征谱秩的耦合。PEFT 中 adapter rank 限制更新子空间，可能改变 FFN 表征谱 scaling；可根据目标层 hard-rank 动态分配 LoRA rank。低算力实验在小 GPT 上做 instruction tuning，对比 uniform rank、spectral-rank rank allocation。预期收益是连接 optimizer scaling 与 PEFT；风险是 hard-rank 指标与下游能力相关性不稳定。

改进点 3：从“优化器比较”推进到 optimizer-architecture co-design。对 FFN expansion ratio、activation、optimizer 组合做小网格，观察哪些结构最能把 optimizer 的谱优势转化为 loss/accuracy。低算力实验用 20M-100M 模型短训，记录 loss、hard/soft rank、rare-token performance。预期收益是给架构设计提供可操作建议；风险是小模型结论外推到大模型有限。

### 10. [The Attribution Impossibility](https://arxiv.org/abs/2605.21492)：共线性下特征归因

原方法事实：论文讨论在 collinearity 下特征 ranking 难以同时满足 faithful、stable、complete，并分析多类模型中的 violation。

改进点 1：从单一 ranking 改成等价特征组归因。既然共线特征间的唯一排序可能不可能，不如输出 feature coalition attribution：先识别近似等价/共线组，再在组内给不确定区间而非强行排序。低算力实验用 sklearn 合成共线数据和 UCI tabular 数据，对比 SHAP/permutation/rank interval。预期收益是规避不可能性结论的一部分；风险是用户仍想要单特征解释。

改进点 2：加入 stability-aware attribution objective。对归因方法增加扰动一致性约束：在 bootstrap、特征旋转、等价模型重训后，归因分布应保持组级稳定。低算力实验训练小 MLP、Lasso、RandomForest，评价 rank variance 和 predictive faithfulness。预期收益是把理论负结果转成可优化指标；风险是稳定性提升可能牺牲局部 fidelity。

改进点 3：设计 attribution abstention 机制。若检测到 Rashomon set 或共线性导致 ranking 不可辨，解释器直接输出“不可判别区域”和最小可区分组，而不是给虚假的确定排名。低算力实验用合成数据控制相关系数从 0 到 0.99，测 abstention calibration。预期收益是提升解释可信度；风险是实际产品场景可能不喜欢“不回答”的解释器。
