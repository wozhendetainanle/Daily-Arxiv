## 低算力可改进的 10 篇：每篇 3 个方法级改进点

下面的事实部分来自 arXiv abs/PDF 首页；改进点、实验设计、收益和风险为推断。目标是小卡、短训练或 toy benchmark 可完成的可证伪实验。

### 1. [Federated LoRA Fine-Tuning](https://arxiv.org/abs/2605.21217)

改进点 1：把 CLAIR 的污染检测从全局低秩分解改成“层级子空间一致性”检测，分别在 attention/MLP LoRA 矩阵上估计共享子空间。低算力实验可用 7B 以下模型或线性 LoRA toy task，模拟 20 个客户端、3-5 个污染客户端。预期收益是定位污染发生在哪些层；风险是层级样本更少，子空间估计方差变大。

改进点 2：加入 rank-adaptive federation。服务器端根据客户端共享奇异谱能量自动给不同层分配 LoRA rank，而不是固定 rank 聚合。实验可用 GLUE/Alpaca 小子集比较固定 rank、全局 rank、层级 rank。预期收益是同等参数量下减少异构任务冲突；风险是 rank 策略可能过拟合当前客户端分布。

改进点 3：加入一轮低成本鲁棒 aggregation baseline，如 trimmed mean/Krum 与 CLAIR 组合。实验只需合成 client update，不需要完整 LLM 训练。预期收益是增强污染客户端极端值鲁棒性；风险是 robust aggregation 可能损伤真实少数任务客户端。

### 2. [SMoA](https://arxiv.org/abs/2605.21147)

改进点 1：做动态谱门控，让 adapter 根据输入 token 或任务标签选择可调谱方向。低算力实验可在 RoBERTa/T5-small 上做多任务微调。预期收益是降低不同任务的谱方向冲突；风险是 gating 学到任务 shortcut。

改进点 2：把 SMoA 与普通 LoRA 做 hybrid：低层用 LoRA，高层用 spectrum modulation。实验比较全 LoRA、全 SMoA、分层 hybrid 的参数-效果曲线。预期收益是把低层稳定性和高层表达力分开；风险是层划分超参增加。

改进点 3：加入 singular-direction dropout，训练时随机屏蔽部分谱方向，测试时集成。低算力可只在小模型上加 mask。预期收益是提高低 rank 泛化；风险是过强 dropout 会削弱 adapter 容量。

### 3. [ChunkFT](https://arxiv.org/abs/2605.21177)

改进点 1：把 chunk 激活调度改成梯度漂移感知，优先更新最近梯度方向变化大的 sub-tensor。实验可在 1B 以下模型上微调指令子集。预期收益是减少无效 chunk 更新；风险是漂移指标噪声大。

改进点 2：加入 chunk-level optimizer state quantization，只给活跃 chunk 保留高精度状态。低算力实验测 8-bit/4-bit state 对 loss 和显存的 Pareto。预期收益是进一步压显存；风险是切换精度导致收敛抖动。

改进点 3：和 LoRA 做 warm-start：先用 LoRA 找敏感层，再把 ChunkFT 预算集中到这些层。实验不需要大模型，7B 以下即可。预期收益是把全参微调预算花在有效位置；风险是 LoRA 敏感层不一定等于全参敏感层。

### 4. [Correcting Stochastic Update Bias](https://arxiv.org/abs/2605.20756)

改进点 1：做 adaptive microbatch split，只在 preconditioner-bias proxy 高时启用 cross-fitting。实验在小 GPT 上比较 always split、never split、adaptive split。预期收益是减少额外计算；风险是 proxy 阈值难设。

改进点 2：对 AdamW/Sophia/Shampoo 分别只修正最主要偏差项，做轻量 ablation。低算力实验可用 100M 级语言模型短训。预期收益是找出最值钱的 correction；风险是不同模型阶段主导偏差不同。

改进点 3：把 bias correction 只应用到高噪声层或 embedding 层。实验结合梯度方差统计做 layerwise 开关。预期收益是降低吞吐损失；风险是层级噪声估计不稳定。

### 5. [HORST](https://arxiv.org/abs/2605.21104)

改进点 1：做 adapter-only HORST，把稀疏几何只施加在 LoRA/adapter 参数上。低算力实验可微调 1B 以下模型。预期收益是把稀疏优化引入 PEFT；风险是 adapter 参数太少时稀疏收益有限。

改进点 2：学习 optimizer composition order，而不是固定算子顺序。实验枚举 3-4 种组合顺序即可。预期收益是判断非交换性是否真带来可用调度；风险是搜索空间扩大。

改进点 3：加入 sparsity warmup schedule，先稳定训练再增强 L1 bias。实验比较固定稀疏强度和 warmup。预期收益是减少早期不稳定；风险是最终稀疏率下降。

### 6. [LamPO](https://arxiv.org/abs/2605.21235)

改进点 1：把 pairwise advantage 与 ACR 诊断结合，当组内 reward 全同或近全同时自动退回 token/span-level credit。低算力实验用 GSM8K 小子集离线 rollouts。预期收益是减少无效梯度；风险是 span credit 估计噪声。

改进点 2：用温度控制 pairwise comparison 的置信权重，训练中根据 KL/entropy 自动调节。实验比较固定温度、线性退火、自适应温度。预期收益是稳定 RLVR；风险是控制器振荡。

改进点 3：把 reference solution 只用于 hard examples，降低人工/教师依赖。实验按 reward margin 选择样本。预期收益是节省标注/教师调用；风险是 hard-example 判定偏差。

### 7. [Equilibrium Reasoners](https://arxiv.org/abs/2605.21488)

改进点 1：增加 attractor early stopping，当 latent state 收敛到稳定 basin 时停止迭代。低算力实验在 Sudoku/Maze/ARC toy tasks 上测 compute-accuracy 曲线。预期收益是节省 test-time compute；风险是错误 attractor 也可能稳定。

改进点 2：加入 basin diversity regularizer，训练时鼓励不同初始化到达相同正确 attractor 但保留早期轨迹多样性。实验比较单轨迹、多轨迹聚合、多样性正则。预期收益是提升 OOD reasoning；风险是正则过强拖慢收敛。

改进点 3：用可视化 toy dynamical system 验证 attractor 假设。小 MLP 或线性递归模型即可。预期收益是把论文机制讲清楚；风险是 toy 结论外推有限。

### 8. [Large-Step Training Dynamics](https://arxiv.org/abs/2605.21292)

改进点 1：把理论中的有效步长参数转成 layerwise LR 警报器，检测接近 catapult/chaos 区域的层。低算力实验用 tiny transformer 做短训。预期收益是给大 LR 训练一个可观测指标；风险是二因子模型过简化。

改进点 2：在 toy transformer 上加入 AdamW/Muon 近似，观察预条件器是否改变相图边界。实验计算量很小。预期收益是连接现代 optimizer；风险是解析性下降。

改进点 3：做 embedding layer 单独学习率扫描，和 hyperparameter transfer 论文交叉验证。预期收益是形成可操作 recipe；风险是数据集依赖强。

### 9. [Combining quantum states and MERA tensor networks](https://arxiv.org/abs/2605.21447)

改进点 1：先做纯经典 shadow-noise ablation，固定 MERA 优化，只改变 classical shadow 样本量。低算力可用小 Ising 链。预期收益是确定量子资源预算下限；风险是小系统不代表大系统。

改进点 2：加入 adaptive boundary tensor selection，不是固定量子态资源，而是根据 MERA 残差选择边界。实验只需模拟不同边界采样。预期收益是提高样本效率；风险是选择策略可能偏向易实例。

改进点 3：和普通 TTN/MPS baseline 做等参数量比较。预期收益是判断 MERA+quantum state 的独立贡献；风险是 baseline 调参不公平。

### 10. [Graph-Based Visual Reasoning](https://arxiv.org/abs/2605.20942)

改进点 1：把 CRS 的 recursive graph queries 移植到简化 ARC/PGM 风格网格任务，验证是否能泛化到非道路抽象视觉推理。低算力实验可合成小图。预期收益是扩大主题相关性；风险是道路图结构和 ARC 结构差异大。

改进点 2：加入 proof trace 输出，让每个 VQA 答案附带 graph query 路径。实验评估 answer accuracy 与 trace faithfulness。预期收益是增强可解释性；风险是 trace 可能只是后验合理化。

改进点 3：用小 VLM + frozen graph encoder 替代大模型，测试结构是否比模型规模更关键。低算力实验可用轻量 CLIP/BLIP。预期收益是形成低资源 baseline；风险是小模型语言能力不足掩盖结构收益。
