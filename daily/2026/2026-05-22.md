## 逐篇要点

1. SeqLoRA：事实：面向多概念 text-to-image 个性化，提出 bilevel 正交 LoRA，并给出收敛与遗忘界。推断：很贴近 LoRA 和 continual learning；局限可能在大规模真实多主体组合与非扩散架构泛化。
2. One LR Doesn't Fit All：事实：用 heavy-tailed self-regularization 为 Transformer 层分配不同学习率，报告最高 1.5x 加速。推断：优化器/训练动力学价值很高；局限是 60M-1B 规模外推到更大 LLM 仍需验证。
3. AI-Driven Formal Proof Search：事实：Google DeepMind 系统用 Lean/LLM 解决部分开放 Erdős 与 OEIS 问题。推断：神经符号与自动数学研究的强信号论文；局限是推理成本和任务选择偏差。
4. Bandit Convex Optimization：事实：研究带梯度预测的 bandit convex optimization，给出两点反馈算法和下界。推断：理论干净，和优化主题高度匹配；局限是实际 ML 训练场景映射仍间接。
5. LABO：事实：把 LLM 低成本预测接入 Bayesian optimization，并给出 regret bound。推断：科学发现/低样本实验很值得跟；局限是 LLM 预测偏差可能污染 BO 后验。
6. Formal Theorem Proving Symmetries：事实：用 category-theoretic rewriting 定义 theorem proving 的 equivariance/invariance，并提出 test-time aggregation。推断：神经符号方向非常有新意；局限是推理预算会增加。
7. DualOptim+：事实：为 LLM unlearning 设计共享/解耦 optimizer states，并提供 8-bit 变体。推断：适合低资源复现；局限是 unlearning 指标和真实删除保证仍要谨慎看。
8. Semantic Entropy Fails：事实：分析语义熵在 policy optimization 中的 anisotropic/calibration gap，提出 GCPO。推断：后训练优化中的 uncertainty signal 很关键；局限是依赖 reward 校准质量。
9. WPO Convergence：事实：证明 entropy-regularized MDP 下 Wasserstein Policy Optimization 可线性收敛。推断：理论价值高；局限是连续时间、正则性假设与实际离散算法仍有距离。
10. Propagation-of-Chaos：事实：给 shallow NN mean-field dynamics 的 uniform-in-time finite-width bound。推断：深度学习理论含金量高；局限是模型较浅、假设偏数学化。
11. GeneCS：事实：面向 stabilizer codes 的 code surgery 编译器，报告显著减少 ancilla/checks。推断：量子纠错工程价值强；局限是实际硬件噪声模型仍需更细验证。
12. QAOA Threshold：事实：给出 2-local QAOA 交互度 2/3 的经典可模拟性与采样困难阈值。推断：能校准“QAOA 优势”叙事；局限是采样困难不等同优化优势，论文也明确指出这一点。
13. HyLoVQA：事实：用 hypernetwork 动态生成 LoRA adapters 做 continual VQA。推断：LoRA + 视觉推理邻近主题；局限是更抽象的 Raven/AVR 推理并未直接覆盖。
14. From Parameters to Data：事实：提出用 task-sensitive attention heads 同时做数据筛选和结构剪枝。推断：和 PEFT/数据选择耦合很有实用性；局限是强依赖 head 重要性的稳定估计。
15. Boundary-Layer Scaling：事实：解释 online softmax classification 中 1/3 scaling 的边界层机制。推断：对 scaling law 理论很有启发；局限是 teacher-student 模型简化明显。
16. Optimizer-Induced Spectral Scaling：事实：比较 AdamW/Muon 等优化器如何改变 FFN 表征谱秩 scaling。推断：把 optimizer 提升为表征 scaling 轴很有价值；局限是谱指标与下游能力因果关系还需补强。
17. Adaptive PID Optimizer：事实：把 AMSGrad/DiffGrad 思路并入 AdaPID，报告更好收敛和稳定性。推断：低算力可复现；局限是基准偏传统，需现代 Transformer 验证。
18. Adiabatic QPE：事实：提出模拟硬件友好的 adiabatic quantum phase estimation，接近 Heisenberg scaling。推断：量子算法基础价值高；局限是实现条件和误差模型需继续看。
19. RL Ion Shuttling：事实：用 RL 优化 trapped-ion quantum computer 的 ion shuttling，报告最高 36.3% 操作减少。推断：量子控制与 RL 交叉有应用味；局限是芯片/任务分布泛化待验。
20. QTT Elliptic Solver：事实：用 Quantized Tensor Trains 做多尺度椭圆方程 full-field simulation。推断：张量网络数值方法相关；局限是非 cs 主类，和 ML 连接较弱。

## 低算力可改进的 3 个方法点

下面不是泛泛的复现实验，而是我认为可以直接改方法、且低算力可验证的三个方向。事实部分来自论文摘要/PDF 首页；改进方案、实验设计和收益风险为推断。

### 1. SeqLoRA：把“正交约束”改成干扰感知的动态 rank/正交预算

原方法事实：[SeqLoRA](https://arxiv.org/abs/2605.22743) 用 bilevel optimization 联合优化 LoRA 两个因子，并用正交适配降低多概念持续学习中的表示干扰。它的强项是把 LoRA basis 从数据里学出来，而不是冻结 basis；问题是，多概念组合时不同层、不同概念的干扰强度并不均匀，统一 rank 或统一正交强度可能会浪费容量。

方法改进：为每个概念-层对维护一个轻量级干扰分数 `I(layer, concept)`，由三项组成：当前概念 LoRA 更新与历史概念 LoRA 更新的子空间夹角、当前 batch 在旧概念验证样本上的 loss drift、以及 cross-attention token 对旧概念 token 的注意力泄漏。训练时不再固定每层 rank 和正交惩罚，而是根据 `I` 动态分配 rank 与正交权重：高干扰层增加正交惩罚或临时扩 rank，低干扰层压缩 rank 或共享 basis。这样改的是 SeqLoRA 的核心适配机制，不是只换数据集。

低算力可行性：不需要训练大扩散模型；可以用 Stable Diffusion 1.5 的少量层 LoRA，只选 10-20 个 DreamBooth/CustomConcept101 概念，每个概念 3-5 张图，分辨率降到 512 或更低。干扰分数可以离线缓存，rank 只在少数候选值里选，比如 `r in {2, 4, 8}`。

具体小实验：基线设三组：固定 rank SeqLoRA、固定 rank 加统一强正交、动态 rank/动态正交。任务设为顺序学习 10 个概念，然后测试单概念保持、多概念组合、旧概念遗忘。指标用 CLIP-I/CLIP-T、DINO identity similarity、旧概念重建相似度、以及多概念 prompt 中 attribute binding error。关键 ablation 是只动态 rank、只动态正交、二者同时动态。

预期收益与风险：收益是用相同或更少参数减少 attribute interference，尤其在容易混淆的相似概念上更明显；风险是干扰分数可能被 CLIP/DINO 这类代理指标带偏，导致 rank 分配过拟合代理指标。若失败，也能给出有价值结论：SeqLoRA 的收益主要来自全局 bilevel basis，而不是 per-layer 容量调度。

### 2. One LR Doesn't Fit All：把 HT-SR layerwise LR 扩展成“谱状态 + 梯度噪声”的双信号学习率控制

原方法事实：[One LR Doesn't Fit All](https://arxiv.org/abs/2605.22297) 用 Heavy-Tailed Self-Regularization 衡量各 Transformer 层权重谱的 heavy-tailedness，并据此给不同层分配学习率。它的核心判断是：不同层结构异质，统一学习率低效；弱 heavy-tail 层给更大学习率，强 heavy-tail 层给更小学习率。

方法改进：HT-SR 只看权重谱，容易漏掉训练过程里的梯度噪声与 batch-level instability。可以把 layerwise LR 改成双信号控制器：`lr_l = base_lr * f(alpha_l) * g(snr_l)`。其中 `alpha_l` 是该层谱 heavy-tail 指标，`snr_l` 是最近若干 step 的梯度信噪比或梯度方向一致性。直觉是：谱上“欠成熟”的层应该学快，但如果该层梯度噪声很大，就不能盲目放大学习率。这个改法会把论文的静态/半静态谱规则推进到在线 optimizer controller。

低算力可行性：只需小语言模型预训练或继续训练，不需要 RLHF。可用 60M-150M GPT、TinyStories/OpenWebText 子集、上下文 512，训练 5k-20k steps。谱指标不必每步算，可以每 200-500 steps 抽样计算 FFN/attention projection 的小型 eigenspectrum；梯度信噪比用 EMA 统计即可。

具体小实验：比较四组：uniform LR、原 HT-SR LLR、只用 gradient SNR 的 LLR、HT-SR + SNR 双信号 LLR。额外加一个 LoRA/QLoRA 微调版本，观察该规则是否能迁移到参数高效微调。指标除 validation loss 和 downstream zero-shot 外，还看训练早期 loss spike 次数、不同层更新范数分布、FFN 表征谱秩变化、以及相同 loss 下的训练 wall-clock。一个很干净的实验是固定总 token budget，看双信号方法是否能减少需要手调 warmup/decay 的次数。

预期收益与风险：收益是比单纯 HT-SR 更稳，尤其在小 batch、低精度、LoRA 微调时可能减少 learning-rate 过冲；风险是控制器引入两个超参，若调参成本吃掉收益，就不够优雅。若实验显示 SNR 只在早期有用，可以退一步提出分阶段策略：warmup 阶段用 SNR 抑制噪声，稳定后主要用 HT-SR 分层。

### 3. Formal theorem proving symmetries：从 test-time rewrite ensemble 改成“预算感知的 rewrite bandit”

原方法事实：[What are the Right Symmetries for Formal Theorem Proving?](https://arxiv.org/abs/2605.22257) 指出 LLM theorem prover 对等价表述非常敏感，并用 rewriting categories 定义 proof equivariance 和 success invariance。论文提出通过等价 rewritings 的 test-time aggregation 来改善鲁棒性。

方法改进：test-time aggregation 的直接问题是预算平均撒出去，很多 rewrite 其实不会增加成功率。可以把 rewrite 选择改成预算感知的 bandit：每个 rewrite family 是一个 arm，例如变量重命名、交换律/结合律重排、目标式左右变换、lemma 引入、归纳变量替换。系统先用少量 probe samples 估计每个 arm 的 proof-progress reward，再把剩余 LLM 调用预算分配给高收益 arm。reward 不只看最终证明成功，还看 Lean 返回的中间信号：错误类型是否变浅、goal 数是否减少、是否生成可复用 lemma、是否进入更短 tactic trace。

低算力可行性：不需要训练模型，只改 test-time search policy。可以用开源小模型或 API 小预算跑 MiniF2F/ProofNet 子集，每题限制 16-64 次采样。Lean 编译反馈是天然监督信号，不需要人工标注。

具体小实验：选 100-200 道 Lean theorem proving 题，固定总采样预算。比较三组：原始 prompt sampling、均匀 rewrite ensemble、rewrite bandit。每组记录 pass rate、平均 Lean calls、成功证明长度、失败题的最深 proof state progress。再做跨主题拆分：number theory、algebra、induction，观察不同 rewrite arm 是否有稳定偏好。如果要更进一步，可以把 bandit 的 arm prior 设为题面特征的线性模型，例如是否出现 commutative operator、是否有 nested expression、是否有 induction variable。

预期收益与风险：收益是用同样推理预算提高成功率，或者用更少预算达到同样成功率；这比单纯“多采样”更像方法改进。风险是 Lean 中间 reward 与最终证明成功不完全一致，可能偏向短期 progress 而错过需要长链规划的证明。若出现这种情况，可以加入 delayed reward 或保留一小部分 uniform exploration，避免过早收敛到局部 rewrite family。
